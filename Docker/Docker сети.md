### Сеть в Docker 
При загрузке **docker** на компьютер, он создаёт виртуальный сетевой мост **bridge** docker0 с ip адресом (172.17.0.1/16)

Каждому контейнеру выдаётся свой виртуальный сетевой интерфейс(veth) типо сетевой карты второй подключён к мосту 


| **bridge**  | частная сеть на одном хосте контейнеры видять друг-друга по именам                      | стандарт для одного сервера       |
| ----------- | --------------------------------------------------------------------------------------- | --------------------------------- |
| **host**    | контейнер юзает сеть хоста напрямую без изоляции. Нет своего IP порты не пробрасываются | максимальная скорость низкий пинг |
| **none**    | полная standalone изоляция                                                              | для задач без сети                |
| **overlay** | одна сеть из нескольких серверов(**кластер Swarm**)                                     | микро сервисы на разных машинах   |

### Комманды для управлениями сетями в докере

```
$ docker network ls
NETWORK ID     NAME      DRIVER    SCOPE
4b6292a59a30   bridge    bridge    local
2cb969eb8040   host      host      local
1e06b6f2bc41   none      null      local
```

```
docker network create my-net          # создать bridge сеть
docker network connect my-net контейнер # подключить контейнер
docker network disconnect my-net контейнер # отключить
docker network inspect my-net         # детали сети
```

```
-d none - выбор драйвера сети
```
