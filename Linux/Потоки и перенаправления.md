### **Redirection**
вlinux можно удобно перенаправлять *источник* или *приёмник* потока с клавиатуры/экрана с помощью 
 ```< >> > ```
#### **Перенаправление stdout** ``` > >> ```
 ```>``` полная перезапись файла (полное удаление содержимого и запись новой информации)
```
rascal@ubuntu:~/test$ cat redirecting
Random text
blablabla
blablabla
rascal@ubuntu:~/test$ echo 'New text' > redirecting
rascal@ubuntu:~/test$ cat redirecting
New text
rascal@ubuntu:~/test$
```

``` >> ``` дозапись к файлу не трогает содержимое а просто дописывает в конец файла 
```
rascal@ubuntu:~/test$ cat redirecting
New text
rascal@ubuntu:~/test$ echo 'blablabla' >> redirecting
rascal@ubuntu:~/test$ cat redirecting
New text
blablabla
rascal@ubuntu:~/test$
```
#### **Перенаправление < stdin** 
``` > ``` используется чтобы передать комманде содержимое файла в качестве ввода заместо клавиатуры
```
rascal@ubuntu:~/test$ sort < redirecting
asdasdad
bvcasdas
cdsasdwe
vaascdss
rascal@ubuntu:~/test$
#отсортировало в алфавит порядке строки
```
#### **Перенаправление stderr (`2>`)**

`2>` перезапись потока ошибок `2>>` дозапись потока ошибок

```
rascal@ubuntu:~/test$ ls no_file.txt 2> errors.log
rascal@ubuntu:~/test$ cat errors.log
ls: cannot access 'no_file.txt': No such file or directory
rascal@ubuntu:~/test$

```
Выкинуть все ошибки в очищаемую директорию (в пустоту)
```
rascal@ubuntu:~/test$ ls no_file.txt 2> /dev/null
```
###  **3. Комбинирование и полный контроль**

Перенаправление обоих потоков (stdout и stderr) в разные места
Можно перенаправить стандартный вывод в один файл, а ошибки — в другой.
```
rascal@ubuntu:~/test$ find / -name "*.conf" 1> output.txt 2> errors.txt
```

#### **Cheet-Sheet**

| Команда            | Что делает                                    |
| ------------------ | --------------------------------------------- |
| `cmd > file`       | stdout в file (перезапись)                    |
| `cmd >> file`      | stdout в file (добавление)                    |
| `cmd 2> file`      | stderr в file (перезапись)                    |
| `cmd 2>> file`     | stderr в file (добавление)                    |
| `cmd < file`       | Взять данные для stdin из file                |
| `cmd > file 2>&1`  | Классика: Оба потока (stdout и stderr) в file |
| `cmd &> file`      | Bash-стиль: Оба потока в file (проще)         |
| `cmd1 \| cmd2`     | stdout cmd1 отправить на stdin cmd2           |
| `cmd1 \|& cmd2`    | stdout и stderr cmd1 отправить на stdin cmd2  |
| `cmd > /dev/null`  | Выкинуть stdout                               |
| `cmd &> /dev/null` | Выкинуть и stdout, и stderr ("режим тишины")  |
