### **Переменные окружения** 
Это именнованные значения, которые хранятся в памяти и доступны всем программам из оболочки. Они влияют на то как работают программы 
`env`- просмотр всех текущих переменных окружения 

```
env # показать все переменные

echo $PATH # показать содержимое конкретной перемен
```

| `HOME`  | Путь к домашней директории юзера     |
| ------- | ------------------------------------ |
| `USER`  | Имя текущ пользователя               |
| `SHELL` | Путь к текущей коммандной оболочке   |
| `PWD`   | Текущая working directory            |
| `PATH`  | Список для поиска исполняемых файлов |
| `LANG`  | Настройки языка (кодировка,языки)    |
`export` - экспорт переменных
При создании переменной в SHELL она доступна только в этом SHELL чтобы сделать её доступной дочерним процессам (программам запуска) нужно её экспортировать

```
MY_VAR="hello" # создать переменную для текущ Shell

export MY_VAR # сделать переменную доступной 

unset MY_VAR # удалить переменную из окружения
```
Пример
```rascal@ubuntu:~$ LOCAL_ONLY="secret"
rascal@ubuntu:~$ bash -c 'echo $LOCAL_ONLY'
# пусто
rascal@ubuntu:~$ export LOCAL_ONLY
rascal@ubuntu:~$ bash -c 'echo $LOCAL_ONLY'
secret

```

`PATH`- самая полезная переменная
Она содержит список директорий разделённых двоеточием, где shell ищет исполняемые файлы  

```
# Добавить ~/scripts в начало PATH (временное, для текущей сессии)
export PATH="$HOME/scripts:$PATH"

# Добавить в конец PATH
export PATH="$PATH:$HOME/scripts"
```
