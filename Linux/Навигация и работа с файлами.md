### **Поиск файлов**
#### `find` - **мастер поиска**

 `find` комманда для поиска файлов и директорий в реальном времени 
 **Базовый синтаксис:** `find [путь] [выражение]`
```
rascal@ubuntu:~/test$ ls
errors.log  errors.txt  output.txt  redirecting
rascal@ubuntu:~/test$ find *redirecting*
redirecting
```

`find . -type f -name "*.txt"`- поиск только по файлам 

```
rascal@ubuntu:~/test$ ls
errors.log  errors.txt  output.txt  redirecting
rascal@ubuntu:~/test$ cd ..
rascal@ubuntu:~$ find . -type f -name "*.txt"
./test/errors.txt
./test/output.txt
```

|                        |                                  |                                                   |
| ---------------------- | -------------------------------- | ------------------------------------------------- |
| `-type d`              | Искать только директории         | `find . -type d -name "project*"`                 |
| `-size [+-]N[c/k/M/G]` | Поиск по размеру                 | `find / -size +100M` (больше 100 МБ)              |
| `-mtime [+-]N`         | Поиск по дате изменения (в днях) | `find . -mtime -7` (изменено за последние 7 дней) |
| `-user имя`            | Поиск по владельцу               | `find /home -user john`                           |
#### `locate` **— молниеносный поиск**

Ищет по заранее созданной базе данных (обновляется обычно раз в сутки). Очень быстрый, но может не найти свежие файлы.

**Преимущества:** Мгновенный результат.  
**Недостатки:** База данных может быть устаревшей, не видит файлы в неиндексированных местах.
```
sudo apt install plocate

rascal@ubuntu:~/test$ locate redirecting
/home/rascal/test/redirecting

rascal@ubuntu:~/test$ sudo updatedb
# обновить базу поиска 
```
**Сравнение:** `find` точный, но медленный (особенно при поиске по всей системе). `locate` быстрый, но может быть неточным из-за устаревшей базы.

### **Дерево катологов**
`tree` - красивое дерево катологов в виде древовидной структуры 

```
rascal@ubuntu:~$ sudo apt install tree

rascal@ubuntu:~$ tree -L 2 # -L глубина дерева
.
└── test
    ├── errors.log
    ├── errors.txt
    ├── output.txt
    └── redirecting

2 directories, 4 files

rascal@ubuntu:~$ tree -L 1 -p # -p показать права
[drwxr-x---]  .
└── [drwxrwxr-x]  test

rascal@ubuntu:~$ tree -L 1 -h # -h показать размер файлов
[4.0K]  .
└── [4.0K]  test

```

### **Ссылки (Links)**
#### **Жёсткая ссылка**
Это другое название того же самого файла с этой же inode,
если даже удалить исходный файл он будет доступен по жёсткой ссылке
#### **Мягкая ссылка**
Это обособленный файл содержащий путь(ссылку до оригинального файла) типо ярлыка windows. Также её можно создавать для папок и даже между разными файловыми системами.

```
rascal@ubuntu:~$ # Создать жесткую ссылку
ln original.txt hardlink.txt

# Создать символическую ссылку (опция -s)
ln -s /путь/к/оригиналу symlink.txt

# Пример с директорией (только символическая)
ln -s /var/log logs

```

#### **File - определение типа файла** 
`file`- определяет по содержимому что за файл

```
rascal@ubuntu:~/test$ file errors.txt
errors.txt: Unicode text, UTF-8 text

rascal@ubuntu:~/test$ file bash.sh
bash.sh: a bin/bash script, ASCII text executable
```
